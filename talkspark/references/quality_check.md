# Quality Check

Use this checklist before final output, especially when the script feels generic.

## L1 Format

- Final script body is one paragraph unless requested otherwise.
- Metadata includes 字数, 预计口播, 标题, 发布文案 unless user asked for body only.
- The user did not ask for multiple versions unless multiple versions are provided.
- No long explanation before the script when the user asked for a deliverable.

## L2 Hook

- First sentence locks a target viewer, topic, person, work, tool, or pain.
- First sentence has a judgment or tension.
- The hook is speakable, not just a written headline.

Weak:

```text
今天我们来聊聊AI时代的个人能力变化。
```

Strong:

```text
我最近越来越觉得, AI时代最占便宜的人, 可能不是最懂技术的人。
```

## L3 Specificity

Every core claim should have at least one concrete support:

- A tool name.
- A person or work name.
- A physical scene.
- A tiny action.
- A before/after contrast.

Weak:

```text
AI提升了人类创造力。
```

Strong:

```text
以前你想做个网页得会前端, 现在你可以把想法丢给Codex, 先搭一个能看的原型。
```

## L4 Human Voice

Check if the script sounds like a person thinking and speaking.

Good signs:

- Has "我" when the idea is personal.
- Has a small scene.
- Has a little surprise, uncertainty, or self-correction.
- Has natural turns like "但问题是", "你发现没有", "这就很有意思".

Bad signs:

- Sounds like a training course.
- Sounds like a brand report.
- Uses broad nouns like 赋能, 矩阵, 生态, 价值闭环 without grounding.
- Every sentence is equally polished.

## L5 Landing

The ending should leave one repeatable line.

Examples:

```text
AI不是替你想象, 它是给想象力接上一双手。
```

```text
很多时候, 我们不是不会用AI, 我们只是还没学会把自己的工作讲清楚。
```

```text
未来最稀缺的, 可能不是会不会操作工具, 而是你每天醒来, 脑子里到底有没有东西想变出来。
```
